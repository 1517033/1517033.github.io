[leetcode] 53.Maximum Subarray

Created2022-03-10|Updated2022-03-13|LeetCode
|Post View:

Problem

Example 1:

Input: nums = [-2,1,-3,4,-1,2,1,-5,4]

Output: 6

Explanation: [4,-1,2,1] has the largest sum = 6.

Example 2:

Input: nums = [1]

Output: 1

Example 3:

Input: nums = [5,4,-1,7,8]

Output: 23

Constraints:

  • 1 <= nums.length <= 105

  • -104 <= nums[i] <= 104

    Follow up: If you have figured out the O(n) solution, try coding another solution using the divide and conquer approach, which is more subtle.

Solutions

Solution one (分治法 Divide and Conquer)

时间复杂度：O(nlogn)

这个分治法的思想类似与二分搜索法，假设最大子序列存在，则仅有三种情况：

  • 在中点左侧[begin:mid]
  • 在中点右侧[mid+1:end]
  • 跨越了中点

Solution two (卡达内算法 Kadane’s Algorithm)

时间复杂度： O(n)

空间复杂度：O(1)

​ 这个算法的基本思想就是从第一个数开始相加，如果前面数的和加上这个数的和比这个数小，则让和等于这个数；否则，让和加上这个数。

Solution three (动态规划 dynamic programming )

时间复杂度： O(n)

空间复杂度： O(1)

​ 这个算法的基本思想就是如果前一个数大于0，则将其加到当前元素上；否则，保持不变。同时比较获取最大值。
